OpenWebUI + Ollama Docker 部署完整指南

手把手教你用 Docker 在本地或 PVE 虚拟机中部署 OpenWebUI，配合 Ollama 实现本地大模型对话。支持 NVIDIA GPU 加速配置。

OpenWebUI Docker Ollama PVE GPU

前置要求

  • Docker 20.10+ 和 Docker Compose
  • （可选）NVIDIA GPU + nvidia-docker
  • 至少 4GB RAM，建议 8GB+
  • 20GB 可用磁盘空间
💡 推荐配置
推荐使用 Docker Compose 管理容器，方便后续更新和备份。单一 YAML 文件即可完成全部配置。

安装 Ollama

Ollama 是本地大模型的运行时，提供简洁的 API 接口。OpenWebUI 通过 Ollama API 调用本地模型。

# Linux/macOS 一键安装
$ curl -fsSL https://ollama.com/install.sh | sh

# 验证安装
$ ollama --version

# 下载第一个模型（推荐从 qwen 或 llama 开始）
$ ollama pull qwen2.5:7b

# 启动 Ollama 服务（后台运行）
$ ollama serve

安装 OpenWebUI

# 创建 docker-compose.yml
$ cat > docker-compose.yml << 'EOF'
version: '3.8'
services:
  openwebui:
    image: ghcr.io/open-webui/open-webui:main
    container_name: openwebui
    restart: unless-stopped
    ports:
      - "8080:8080"
    environment:
      - OLLAMA_BASE_URL=http://host.docker.internal:11434
    extra_hosts:
      - "host.docker.internal:host-gateway"
    volumes:
      - openwebui-data:/app/backend/data
    networks:
      - openwebui-net

networks:
  openwebui-net:
    driver: bridge

volumes:
  openwebui-data:
EOF

# 启动
$ docker compose up -d

# 访问 http://localhost:8080

GPU 加速配置

如果使用 NVIDIA GPU，需要配置 Docker 的 GPU 支持：

# 确认 nvidia-docker 已安装
$ docker run --gpus all nvidia/cuda:12.1-base nvidia-smi

# 在 docker-compose.yml 中添加 gpu 配置
$ cat >> docker-compose.yml << 'EOF'
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: all
              capabilities: [gpu]
EOF

$ docker compose up -d
⚠️ PVE 虚拟机注意事项
PVE 虚拟化环境需要开启 GPU 直通（PCIe Passthrough），并在虚拟机配置中添加 PCI 设备。Docker 容器内的 GPU 访问依赖宿主机层面的直通配置。

PVE 虚拟机配置

在 PVE 中创建 Windows/Linux VM 后，GPU 直通步骤：

  1. 在 PVE 宿主机编辑 /etc/modprobe.d/blacklist.conf 屏蔽默认驱动
  2. 在 VM 的 Hardware → Add → PCI Device 选择你的 GPU
  3. 启动 VM，安装对应驱动后 Docker 即可识别 GPU

远程模型调用

如果 GPU 机器作为服务器，其他机器通过 API 调用：

# 在 Ollama 机器上设置环境变量允许外部访问
$ export OLLAMA_HOST=0.0.0.0
$ export OLLAMA_ORIGINS="*"
$ ollama serve

# 客户端机器配置 OpenWebUI 环境变量
$ export OLLAMA_BASE_URL=http://YOUR_SERVER_IP:11434

常见问题

Q: 启动后提示 "Failed to connect to Ollama"

A: 确认 Ollama 已在运行（ollama serve），并且 OLLAMA_BASE_URL 指向正确地址。Windows/Mac 上 Docker Desktop 使用 host.docker.internal，Linux 服务器使用实际 IP。

Q: 模型下载很慢

A: 使用镜像加速：export OLLAMA_MODELS=/path/to/models 配合国内镜像站下载。
