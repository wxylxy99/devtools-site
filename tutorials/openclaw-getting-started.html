OpenClaw 是什么？

一个运行在你服务器上的自主 AI Agent，支持多平台消息接入

OpenClaw 是一个开源的本地 AI Agent 框架，由 Nous Research 团队开发维护。 与传统的聊天机器人或 IDE 插件不同，OpenClaw 是一个持久运行在你自有服务器上的自主智能体—— 它能记忆学习、调用工具、执行自动化任务，并通过 Telegram、Discord、Slack 等多个平台与你保持联系。

OpenClaw 的核心设计理念是「成长性」：你运行它的时间越长，它对你的项目和环境就越了解， 能帮你处理的事情也就越多。它会自动生成并保存 Skills（技能），在 ClawHub 社区与他人共享， 形成一个持续进化的 AI 助手生态。

核心能力一览

多平台消息网关

同时接入 Telegram、Discord、Slack、WhatsApp、Email，一个 Agent 对接所有渠道。

持久记忆与技能学习

自动记录对话历史和执行轨迹，生成可复用的 Skills，越用越聪明。

自然语言定时任务

用日常语言描述定时任务，OpenClaw 自动转换成 cron 表达式并执行。

沙箱隔离执行

支持 Docker / SSH / Singularity / Modal 等多种隔离环境，保护宿主机安全。

子 Agent 并行执行

派生出独立子 Agent 处理复杂任务，互不干扰，支持 Pipeline 串联。

40+ 内置工具

网页搜索、浏览器自动化、终端命令、代码执行、图像生成、语音合成等开箱即用。

💡 与 Hermes Agent 的关系： OpenClaw 是 Hermes Agent 项目在本地部署场景下的品牌名称， 两者共享同一套代码基础。OpenClaw 专注于 Docker 化本地部署和轻量入门体验， 特别适合不想手动管理 Python 环境、追求快速启动的用户。

环境准备：安装依赖

在开始之前，需要确保你的系统满足基本要求并安装好必要软件

系统要求

项目 最低要求 推荐配置
操作系统 Ubuntu 20.04+ / macOS 12+ / WSL2 Ubuntu 22.04 LTS
内存 8 GB RAM 16 GB RAM 或更多
磁盘空间 10 GB 可用 20 GB SSD
网络 能访问 OpenAI/OpenRouter API 稳定的翻墙线路或国内 API 镜像

Step 1: 安装 Docker

Docker 是 OpenClaw 的推荐运行方式。如果你的系统还没有 Docker，执行以下步骤：

bash 
# 一键安装 Docker（Ubuntu/Debian）
curl -fsSL https://get.docker.com | sh

# 将当前用户加入 docker 组（避免每次 sudo）
sudo usermod -aG docker $USER
newgrp docker

# 验证安装
docker --version
# 输出示例: Docker version 24.0.7, build afdd53b

⚠️ 注意： 在 macOS 上推荐使用 Docker Desktop， 安装后确保 Docker Engine 处于运行状态。Windows 用户请务必通过 WSL2 模式安装 Docker Desktop， 勿直接在 PowerShell 中运行容器。

Step 2: 安装 Docker Compose

Docker Compose 让我们可以用一个 YAML 文件管理所有服务（Agent、Gateway、数据库等）的生命周期。 大部分场景下 OpenClaw 只需要 docker compose 命令即可。

bash 
# 检查是否已内置（Docker Desktop 和新版 docker 包自带）
docker compose version
# 如果报错 command not found，手动安装：
# https://docs.docker.com/compose/install/

Step 3: 准备 API Key

OpenClaw 本身不托管大模型，你需要提供一个 API 端点来调用 LLM。 目前支持的来源包括：

  • Nous Portal（推荐，免费额度，需注册 portal.nousresearch.com）
  • OpenRouter（openrouter.ai，支持数百个模型，按量计费）
  • OpenAI API（官方 API，需要境外支付方式）
  • 自定义 OpenAI 兼容端点（如 Groq、LocalAI、Ollama 等）

✅ 推荐新用户： 前往 Nous Portal 注册账号， 在仪表盘复制你的 API Key，完全免费，无需信用卡。

Step 4: 防火墙与网络

OpenClaw 的控制面板默认监听 3000 端口，Gateway 默认监听 8000 端口。 如果你的服务器在云上，需要在云控制台的安全组中放行这些端口。

bash 
# Ubuntu/Debian 使用 ufw 放行端口（如果你用 ufw）
sudo ufw allow 3000/tcp
sudo ufw allow 8000/tcp
sudo ufw reload

Docker 一键部署

一条命令，在任何支持 Docker 的系统上启动完整的 OpenClaw

方式一：快速启动（推荐新手）

这是最快的上手方式。只需要一条命令，OpenClaw 会自动拉取镜像、创建配置文件目录、 启动所有容器服务。整个过程通常在 2-3 分钟内完成。

bash 
# 一键启动 OpenClaw（自动下载最新版本）
curl -fsSL https://openclaw.ai/install.sh | docker compose -f - up

⚠️ 首次运行会拉取多个 Docker 镜像（约 3-5 GB）， 请确保网络畅通。如果拉取缓慢，可以使用国内镜像加速： 在 /etc/docker/daemon.json 中加入 "registry-mirrors": ["https://docker.mirrors.ustc.edu.cn"]， 然后执行 sudo systemctl restart docker。

方式二：手动配置文件启动（推荐有经验用户）

如果你想自定义数据目录、端口、环境变量，建议手动创建 docker-compose.yml 文件。 这样更方便后续升级和数据迁移。

bash 
# 创建项目目录并下载配置文件
mkdir -p ~/openclaw && cd ~/openclaw
curl -fsSL https://openclaw.ai/docker-compose.yml -o docker-compose.yml

然后创建 .env 环境变量文件：

~/openclaw/.env 
# 创建 .env 文件
cat > .env << 'EOF'
# API 配置
OPENCLAW_API_PROVIDER=openrouter
OPENCLAW_API_KEY=your-api-key-here
OPENCLAW_MODEL=anthropic/claude-3.5-sonnet

# 控制面板
OPENCLAW_UI_PORT=3000
OPENCLAW_UI_USERNAME=admin
OPENCLAW_UI_PASSWORD=change-me-before-running

# Gateway
OPENCLAW_GATEWAY_PORT=8000

# 数据持久化
OPENCLAW_DATA_DIR=./data
EOF

💡 模型选择建议： 如果使用 OpenRouter，OPENCLAW_MODEL 可以填入任意支持的模型 ID， 如 anthropic/claude-3.5-sonnet、openai/gpt-4o、 google/gemini-2.0-flash-exp 等。推荐从 Claude 3.5 Sonnet 开始， 它的工具调用能力是目前最强的。

启动与停止

bash 
# 启动所有服务（后台运行）
cd ~/openclaw && docker compose up -d

# 查看运行状态
docker compose ps

# 查看实时日志
docker compose logs -f

# 停止所有服务
docker compose down

# 完全清理（包括数据卷，慎用）
docker compose down -v

验证是否运行成功

启动后等待约 30 秒，让各服务完成初始化。然后访问控制面板确认状态：

bash 
# 检查控制面板健康状态
curl -s http://localhost:3000/api/health | python3 -m json.tool

# 预期输出示例：
# {
#   "status": "ok",
#   "version": "1.2.4",
#   "agent": "connected",
#   "gateway": "connected"
# }

✅ 恭喜！ 如果看到 "status": "ok"，说明 OpenClaw 已成功运行。 现在打开浏览器访问 http://你的服务器IP:3000，输入用户名密码即可进入控制面板。

Telegram / Discord 多渠道接入

让你的 Agent 同时在多个平台接收消息，不遗漏任何一条

OpenClaw 的 Gateway（网关）服务负责对接各个消息平台。一个运行中的 OpenClaw 实例 可以同时连接 Telegram 和 Discord，也可以按需添加更多渠道。 所有渠道的消息都会汇聚到同一个 Agent，会话历史统一管理。

Telegram Bot 接入

Telegram 是接入最简单、体验最流畅的平台。需要先创建一个 Telegram Bot：

  1. 在 Telegram 中搜索 @BotFather，发送 /newbot
  2. 给 Bot 起名字（显示名）和用户名（以 bot 结尾，唯一）
  3. BotFather 会返回一个 HTTP API Token，格式类似 123456789:ABCdefGHIjklMNOpqrSTUvwxyz
  4. 在 .env 或控制面板中填入这个 Token 即可
~/openclaw/.env（追加以下内容） 
# 在 .env 中追加 Telegram 配置
cat >> .env << 'EOF'

# Telegram
TELEGRAM_BOT_TOKEN=your-telegram-bot-token
# TELEGRAM_ALLOWED_USERS 填入你的数字用户 ID（不是用户名），
# 可通过 @userinfobot 获取
TELEGRAM_ALLOWED_USERS=123456789
EOF

⚠️ 重要 — 访问控制： 务必设置 TELEGRAM_ALLOWED_USERS 为你的数字 ID， 而不是留空。OpenClaw 默认不限制访问者，如果不设白名单，任何知道 Bot 用户名的人都可以与你的 Agent 对话， 可能产生意外费用或安全风险。

获取用户 ID 的方法：在 Telegram 搜索 @userinfobot 并发送任意消息，它会回复你的数字 ID。

Discord Bot 接入

Discord 接入稍微复杂，需要在 Discord Developer Portal 创建一个 Application 并开启 Bot 权限：

  1. 前往 Discord Developer Portal， 点击 New Application，起个名字
  2. 在左侧菜单选择 Bot，点击 Add Bot
  3. 在 Privileged Gateway Intents 中开启： PRESENCE INTENT、SERVER MEMBERS INTENT、MESSAGE CONTENT INTENT
  4. 点击 Reset Token 获取 Bot Token（只显示一次，请妥善保存）
  5. 在 OAuth2 > URL Generator 中，勾选 bot scopes 和 Read Message History、Send Messages 权限，生成邀请链接
  6. 打开邀请链接，将 Bot 添加到你的服务器
~/openclaw/.env（追加以下内容） 
# 在 .env 中追加 Discord 配置
cat >> .env << 'EOF'

# Discord
DISCORD_BOT_TOKEN=your-discord-bot-token
# DISCORD_GUILD_ID 为你的服务器（Guild）数字 ID，
# 在 Discord 开发者模式中右键服务器 -> 复制 ID
DISCORD_GUILD_ID=123456789012345678
EOF

💡 获取 Discord Guild ID： 在 Discord 设置中开启 开发者模式（Advanced > Developer Mode），然后右键你的服务器名， 选择 复制服务器 ID。

重启服务使配置生效

bash 
# 重启 OpenClaw 以加载新配置
cd ~/openclaw && docker compose down && docker compose up -d

# 查看 Gateway 日志确认渠道连接状态
docker compose logs -f gateway

与 Agent 对话

配置完成后，在 Telegram 中找到你的 Bot 并发送一条消息，Agent 应该在几秒内回复。 在 Discord 中，你需要在文本频道 @mention Bot 或者直接发消息（取决于配置）， Bot 同样会回复。OpenClaw 的上下文会自动携带之前对话的历史记录。

✅ 多渠道测试： 你可以在 Telegram 发起一段对话，然后切换到 Discord 继续。 对 Agent 来说，这是同一个会话，因为它在服务端维护了统一的记忆和状态。

Web 控制面板详解

通过图形界面管理你的 Agent、渠道、Skills 和定时任务

打开浏览器访问 http://服务器IP:3000，使用 .env 中配置的 OPENCLAW_UI_USERNAME 和 OPENCLAW_UI_PASSWORD 登录。 首次登录后建议立即修改密码。

仪表盘（Dashboard）

登录后默认进入仪表盘，这里展示了 Agent 的整体运行状态：

  • Agent 状态 — 当前是否在线、已运行时长、当前模型
  • 消息统计 — 今日/本周/本月处理的消息数量
  • 活跃渠道 — Telegram、Discord 等渠道的连接状态指示灯
  • 最近活动 — Agent 最近执行的命令、生成的 Skills、触发的定时任务

对话管理（Conversations）

这里列出了 Agent 所有历史会话。你可以选择任意一个会话继续对话， 也可以查看完整的对话轨迹，了解 Agent 在某个问题上做了哪些思考和工具调用。 对话支持导出为 JSON 或 Markdown 格式，方便存档或分享。

Skills 市场（Skills Hub）

OpenClaw 的 Skills 是可插拔的任务模块。一个 Skill 可以是一个文档处理流程、 一个代码审查规则、一个定时报告生成器，甚至是一个完整的自动化工作流。

内置 Skills： OpenClaw 开箱自带 40+ Skills，覆盖 GitHub PR 审查、 数据分析报告、新闻摘要、定时提醒等常见场景。

社区 Skills： 从 ClawHub 或 AgentSkills.io 可以直接安装社区贡献的 Skills。 在 Skills Hub 页面搜索关键词，点击安装即可，无需手动配置。

Skill 安装示例 
# 也可以通过命令行安装 Skill
docker compose exec agent openclaw skills install github-pr-reviewer

# 查看已安装的 Skills
docker compose exec agent openclaw skills list

# 卸载 Skill
docker compose exec agent openclaw skills uninstall github-pr-reviewer

定时任务（Cron Jobs）

OpenClaw 支持用自然语言描述定时任务，Agent 会自动帮你转换成 cron 表达式并按计划执行。 在控制面板的 Cron Jobs 页面，你可以：

  • 创建新的定时任务（如「每天早上 9 点给我发一份天气报告」）
  • 查看所有已创建任务的执行历史和结果
  • 手动触发任意定时任务（测试用）
  • 启用/禁用单个任务，而不必删除
定时任务示例 
# 通过自然语言创建定时任务
docker compose exec agent openclaw cron create "每天早上9点给我发天气报告" \
  --time "09:00" \
  --timezone "Asia/Shanghai" \
  --skill weather-briefing

系统设置（Settings）

在 Settings 页面可以调整以下配置：

配置项 说明
OPENCLAW_MODEL 当前使用的 LLM 模型，可随时切换
temperature 模型温度参数，0-2 之间，越高越有创造性
max_tokens 单次回复最大 token 数，建议 4096
system_prompt 系统提示词，决定 Agent 的角色定位和行为风格
allowed_tools 允许 Agent 调用的工具列表，安全性控制
rate_limit 每个用户每分钟最大消息数，防止滥用

💡 进阶技巧： 如果你想让 Agent 以特定角色运行（比如「代码审查助手」或「产品经理」）， 修改 system_prompt 即可。例如： "你是一个资深的后端架构师，擅长审查 Go 语言代码，关注性能、安全和可维护性。" Agent 会自动调整它的回答风格和关注点。

常见问题与解决方案

遇到问题先查这里，90% 的问题都可以通过以下方法解决

1. Docker 容器启动后立即退出，日志显示 "permission denied"

这通常是因为数据目录权限问题。OpenClaw 的某些服务需要写入 ./data 目录，而当前用户可能不是目录所有者。 解决方式：sudo chown -R $(id -u):$(id -g) ~/openclaw/data， 然后重新 docker compose up -d。 如果使用了 WSL2，确保 Windows 侧的文件夹没有开启 "只读" 或 "同步" 特性。

2. 镜像拉取超时，速度极慢

使用国内 Docker 镜像加速。在 /etc/docker/daemon.json 中添加： {"registry-mirrors": ["https://docker.mirrors.ustc.edu.cn", "https://hub-mirror.c.163.com"]}， 然后 sudo systemctl restart docker。 或者直接在国内云服务器（如腾讯云、阿里云北京区域）上运行，拉取速度会快很多。

3. Telegram Bot 不响应消息

按以下顺序排查：① Bot Token 是否正确填写在 .env 中；② 重启后配置是否生效（docker compose restart gateway）； ③ 在 Bot Father 中发送 /mybots 查看 Bot 状态； ④ 检查 docker compose logs gateway 中是否有 Telegram 连接错误； ⑤ 确认 TELEGRAM_ALLOWED_USERS 中包含你的数字 ID（不是 @用户名）。

4. Discord Bot 显示 "Message Content Intent" 权限不足

这是最常见的 Discord 接入问题。在 Discord Developer Portal 的 Bot 页面， 找到 Privileged Gateway Intents，确保 MESSAGE CONTENT INTENT 已开启（滑块变绿）。 修改后需要点击 Save Changes，然后在 OAuth2 > URL Generator 重新生成邀请链接，将 Bot 重新加入服务器。

5. API 调用报 401 或 403 错误

401 表示 API Key 错误或过期；403 表示 Key 有效但没有访问该模型的权限。 解决方法：登录对应的 API 提供者网站检查 Key 是否正确、 账户是否有余额、是否有权限访问你指定的模型。 如果使用 OpenRouter，可以在仪表盘查看具体模型的可用性。

6. Agent 回复速度很慢（超过 30 秒）

可能的原因：① 模型推理本身较慢（换用更快的模型如 GPT-4o Mini 或 Gemini Flash）； ② 网络延迟（你的服务器到 API 服务商的网络质量差，建议用 curl -w "%{time_total}" 测试）； ③ OpenClaw 工具调用链过长（减少每次对话的工具调用数量）； ④ Docker 容器资源不足（增加 docker-compose.yml 中 agent 服务的内存限制）。

7. 如何升级到新版本？

OpenClaw 使用 Docker 镜像发布，升级非常方便： docker compose pull && docker compose up -d。 这会拉取最新镜像并重启服务。由于数据持久化在 ./data 目录， 升级不会丢失历史对话和 Skills。如果是大版本升级（如 v1.x 到 v2.x）， 建议先阅读 Release Note 中的 Breaking Changes 部分。

8. 如何查看详细日志进行调试？

使用 docker compose logs -f [service_name] 查看指定服务的日志。 主要服务包括：agent（主 Agent）、gateway（消息网关）、 ui（控制面板）。使用 -f 参数可以实时跟踪日志输出。 如果日志太多，可以用 docker compose logs --tail=100 agent 只看最近 100 行。

9. 如何备份和迁移 OpenClaw？

OpenClaw 的所有数据都在 ~/openclaw/data 目录中。 完整备份只需要：tar -czvf openclaw-backup.tar.gz ~/openclaw/data。 迁移到新服务器时，复制这个压缩包，在新服务器上安装 Docker 和 Docker Compose， 创建相同结构的目录，解压后执行 docker compose up -d 即可。 注意确保 .env 文件也被一并复制（API Key 等敏感信息不要泄露）。

10. 控制面板打不开，显示 502 Bad Gateway

这说明 UI 服务没有正常启动。执行 docker compose ps 查看各服务状态， 如果 UI 服务状态不是 "Up"，执行 docker compose logs ui 查看具体错误。 常见原因：端口 3000 被占用（sudo lsof -i :3000 查进程）、 内存不足导致 OOM（dmesg | grep -i oom 查看）。 解决后 docker compose restart ui。
